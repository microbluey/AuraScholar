# AuraScholar 界面设计审查与重设计方向

审查日期：2026-07-14  
审查范围：桌面壳、文献库、学术检索、PDF 阅读器、闪卡、写作素材、检索哨兵、学术主页、设置，以及移动端响应式表现。

## 结论

AuraScholar 的功能架构已经超过“工具原型”阶段，但视觉仍停留在“功能完整的管理后台”。当前最大问题不是颜色不够漂亮，而是缺少稳定的视觉主次：边框、卡片、按钮、标签、提示条和统计卡片几乎都以相近强度出现，用户很难在一眼内判断“当前任务、主要对象、下一步动作”。

建议把产品统一为一个「研究工作台」系统：深色应用框架负责导航和身份，浅色工作面负责浏览与整理，右侧上下文面板负责当前论文的 AI、附件、批注和状态。阅读器切换到专注的 Nocturne 工作区，但沿用同一套组件语义。

## 面向年轻学者的 Neo Academia 方向

年轻化不等于加入更多装饰、卡通图标或游戏化积分。AuraScholar 的目标用户需要的是“数字原生、聪明、快速、有个人风格”，同时仍能在论文阅读和写作场景中保持专注。

- 品牌从单一学院青绿升级为紫色 × 薄荷绿双主色，形成更鲜明的科技与创造力识别。
- 页面标题从后台式模块名称转向有行动感的研究语言，例如“让研究灵感持续生长”。
- 使用轻柔的 Aurora 渐变、20px 面板圆角和更明显但克制的浮层关系，增强现代桌面应用质感。
- 选中态、AI 能力和主操作可以更鲜明；论文正文、元数据和长时间阅读区域继续保持低刺激。
- 强调“专注模式、阅读目标、连续研究流”等年轻研究者真实需要的动机设计，但避免排行榜、虚拟币和强制打卡。
- Nocturne 从传统蓝灰技术风调整为深靛紫研究空间，只在当前工具、AI 页签和关键动作上使用高饱和渐变。

## 当前做得好的部分

- 功能信息架构完整，文献发现、入库、阅读、批注、AI、闪卡、写作素材、监控和发布已经形成闭环。
- 文献库的表格语义比“每篇论文一张卡片”更适合高频扫描。
- Dawn 的冷白底与低饱和青绿色方向是对的，适合长时间阅读。
- 预览模式、状态反馈、键盘快捷键和移动端适配都被认真考虑，产品可信度较高。
- 页面已有明确的领域语言，例如 OA PDF、AI 重点、收录状态、证据快照、FSRS 复习，而不是通用后台术语。

## 主要问题

### 1. 视觉层级过平

当前页面大量使用 1px 边框、浅色卡片和圆角控件。每个区域都“被框起来”，但没有真正的主舞台。学术检索页里顶部统计卡、检索控制台、排序栏、结果卡和右侧详情卡同时抢视线；学术主页页也存在相同问题。

调整方向：减少 35%–45% 的可见边框，用背景分层、留白和选中态替代。一个页面只允许一个高对比主区域和一个主按钮。

### 2. 页面身份不稳定

文献库像数据库表格，检索页像数据看板，主页页像表单后台，阅读器又是独立工具。它们分别成立，但放在同一个产品里缺少共同的骨架。

调整方向：所有桌面页面固定为「应用导航 / 页面命令区 / 主工作面 / 上下文面板」四层。不同功能只替换主工作面和右侧上下文内容。

### 3. 操作优先级不够明确

“导入 PDF”“导入文献库”“添加文献”“刷新”在文献库顶栏拥有接近的视觉重量。检索结果中的来源、可信度、OA、入库动作也密集并列。用户必须逐个阅读控件，而不是通过视觉位置和颜色直接理解下一步。

调整方向：每个场景只保留一个实心主动作；其余动作收纳为次级按钮或菜单。文献库主动作是“添加文献”，检索结果主动作是“加入文献库”，阅读器主动作由当前选区决定。

### 4. 状态标签过多且语义分散

标签、来源、阅读状态、可信度、OA、AI、哨兵等都使用相似的小胶囊，颜色数量较多，用户难以建立稳定含义。

调整方向：只保留四类语义色：青绿=当前/可执行，绿色=完成/可用，琥珀=等待/需确认，紫色=AI/自动生成。普通标签和来源使用中性色。

### 5. 字体系统缺乏“学术编辑感”

目前绝大多数文字采用相近字重和紧凑尺寸，论文题目、页面标题、元数据与操作说明的区分依赖字号而非排版角色。高级感因此更接近 SaaS 后台，而不是研究产品。

调整方向：操作界面继续使用无衬线字体；论文题目、引文预览和个人主页成果列表可以使用克制的衬线字体。正文行高提高到 1.5–1.6，元数据使用更安静的灰阶。

### 6. 右侧上下文能力没有成为系统核心

文献库把工作台放在列表下方，选中论文后视线需要从表格向下跳；检索页已经有右侧详情，但信息组织仍偏“字段堆叠”。阅读器的批注、翻译、AI 和引文关系则天然适合上下文面板。

调整方向：建立统一的 360–400px Context Panel。文献库展示论文摘要、阅读进度、附件、AI、闪卡、哨兵；检索页展示证据和入库决策；阅读器展示批注、翻译、AI 重点和引文。

### 7. 响应式是“可用”，但还不是移动体验

移动端文献库中，大量筛选按钮折成多行，首屏接近一堵控件墙；论文从表格转成卡片后，底部导航进一步压缩了有效高度。桌面信息的简单折行不能形成移动端任务流。

调整方向：移动端首屏只保留搜索、当前视图和筛选入口；筛选放入底部抽屉。论文卡只显示题目、作者、年份、一个主要状态，次要元数据进入详情页。

### 8. 预览提示过于抢眼

预览模式提示条横跨页面且使用暖色背景，它的视觉权重高于真实任务。对演示环境有帮助，但会削弱产品第一印象。

调整方向：缩成顶栏右侧的“预览数据”状态按钮，首次进入时使用一次性轻提示，详细说明放入弹层。

## 新的视觉系统

### 结构

- 应用导航：224px 深墨绿色，承载品牌、模块和账户状态。
- 页面命令区：64–72px，包含全局搜索、快速入库、同步和页面主动作。
- 主工作面：中性浅灰背景上的单一主要表面，列表优先，避免卡片瀑布。
- 上下文面板：360–400px 固定宽度，随当前选中对象更新。
- 状态栏：只保留本地优先、同步和后台任务三个系统级状态。

### 色彩

- Ink：`#12201D`，导航和高对比文字。
- Canvas：`#F3F6F4`，应用工作区背景。
- Paper：`#FFFFFF`，主内容表面。
- Teal：`#2F7D73`，主动作和当前选中。
- Mint：`#DDEEE9`，柔和选中背景。
- Amber：`#C48632`，等待和需确认。
- Violet：`#7559B8`，AI 生成内容。
- Red：仅用于删除、失败和高风险操作。

### 默认明暗策略

- 默认采用 Light-first：雾白导航、淡紫灰画布、白色工作面，适合白天和长时间研究。
- 现代感来自排版、间距、渐变焦点和交互反馈，不依赖大面积深色背景。
- 阅读器默认使用暖灰工作区承托纸张，避免纯白全屏带来的眩光，也避免暗色界面的压迫感。
- Nocturne 保留为用户主动选择的夜间主题，不作为默认品牌展示。

### 排版与尺寸

- 页面标题：28–32px / 650。
- 对象标题：15–17px / 600，论文标题允许两行。
- 正文：14px / 1.55。
- 元数据：12px / 1.45，颜色低于正文两个层级。
- 控件高度：36px；主命令输入 44px；触控目标最小 44px。
- 圆角：控件 8px，面板 12px；胶囊仅用于真正的状态。
- 阴影：只用于浮层和右侧上下文面板，普通分组不使用阴影。

## 各页面重设计重点

### 文献库

- 顶部只保留一个可搜索、可粘贴 DOI/arXiv/URL 的命令输入，以及“添加文献”主按钮。
- 左侧集合与标签保留在应用导航的二级区，不再与主导航竞争。
- 主列表固定列宽，行内只展示题目、作者、年份、来源和两个最重要状态。
- 选中论文直接更新右侧上下文面板，不再把工作台放在列表下方。
- 批量操作仅在勾选后出现，默认界面保持安静。

### 学术检索

- 搜索输入成为页面视觉焦点；数据源状态收纳到输入下方一行。
- 结果默认使用紧凑列表，摘要按需展开。
- 可信度从“彩色标签”升级为证据摘要：命中源数量、DOI 一致性、OA 状态。
- 右侧详情聚焦“为什么值得入库”和“入库后会发生什么”。

### PDF 阅读器

- 作为旗舰体验，进入后隐藏完整应用导航，仅保留返回文献库和论文上下文。
- 页面中央最大化 PDF；批注工具浮动在页面边缘，选区出现上下文工具条。
- 右侧使用“批注 / 翻译 / AI 重点 / 引文”四个固定页签。
- Nocturne 中避免大面积青色描边和泛光，只对焦点与当前页使用青色。

### 闪卡

- 顶部只显示今日队列、连续天数和预计时间。
- 主区域一次只呈现一张卡，来源论文作为弱化上下文。
- 四级评分使用固定位置和键盘映射，颜色只在评分后反馈。

### 检索哨兵

- 以论文为行，以 Accepted → Registered → Online → Issue → Indexed 为横向进度。
- 右侧展示选中节点的证据快照、来源、时间与可信度。
- “立即检查”是次级动作，异常和逾期才使用醒目颜色。

### 学术主页

- 左侧从长表单改为分区编辑器，右侧预览保持固定。
- 成果选择采用可排序的出版物列表，支持置顶和分组。
- 导出与发布动作统一放在页面右上，不与资料编辑混在一起。

### 写作素材与设置

- 写作素材按论文和主题双维度组织，突出“复制引用”“回到原文”和“插入写作项目”。
- 设置使用左侧设置目录和右侧单列内容；连接测试、保存状态与敏感信息提示形成统一模式。

## 实施顺序

1. 先重构设计令牌、排版层级、按钮优先级和状态色，不改业务逻辑。
2. 建立统一的 App Shell、Page Command Bar、Context Panel 和 Status Bar。
3. 先改文献库与阅读器，这两页决定产品高级感的 70%。
4. 将学术检索和检索哨兵迁移到同一上下文面板模式。
5. 最后统一闪卡、学术主页、写作素材与设置。
6. 移动端单独设计任务流，不再直接压缩桌面布局。
